AVETI TRANSCRIPTS - COMPLETE BUILD NOTES
========================================

Project location:
C:\Users\27sum\Downloads\grammar

Current app name:
Aveti Transcripts

Main purpose:
This project turns Khan Academy courses and YouTube playlists into a searchable
transcript library. A user can submit a source URL, the backend creates a
background scrape job, the worker stores courses/playlists, videos, transcripts,
timestamped segments, and transcript embeddings in a database, and the UI lets
users browse, search, export, and read the transcript library.


1. ORIGINAL MVP IDEA
--------------------

The MVP was built around the functional requirement:

- Accept a Khan Academy course URL.
- Scrape course structure.
- Find units, lessons, and videos.
- Fetch available transcripts/subtitles.
- Store everything in a database.
- Show progress through background jobs.
- Provide a UI to browse courses and videos.
- Provide a way to search transcripts.
- Export data.

During the build, the MVP expanded to also support:

- YouTube playlist scraping.
- Categories.
- Admin-only category creation.
- YouTube transcript fetching.
- Transcript embeddings.
- PostgreSQL and pgvector hybrid search.
- Public VM deployment without Docker.

7. PGVECTOR AND HYBRID SEARCH
-----------------------------

Hybrid search was added as an advanced search feature.

Goal:
Search should return videos/transcript chunks by meaning and relevance, not only
exact word matching.

Implementation:
- Transcript text is split into chunks.
- Each chunk gets an embedding and is stored in transcript_embeddings.
- PostgreSQL full-text search is used for lexical relevance.
- pgvector cosine similarity is used for vector similarity.
- Results are combined into a hybrid score.

Files:
- app/services/embeddings.py
- app/services/pgvector_search.py
- app/routes/api.py
- app/database.py
- app/importer.py
- app/services/job_processor.py
- app/services/youtube_backfill.py

Behavior:
- If PostgreSQL has pgvector installed, /api/search uses hybrid search.
- If pgvector is not installed, the app logs a warning and falls back to basic
  keyword search.
- SQLite always uses the fallback keyword search.

Important caveat:
The current embeddings are local deterministic MVP embeddings. They make the
pgvector pipeline work, but true semantic quality would improve if replaced
with real AI embeddings later.

VM warning seen:
extension "vector" is not available

Meaning:
The app is running, but pgvector is not installed on that PostgreSQL server.

Install attempt:
sudo apt install -y postgresql-16-pgvector

If the package is not available, pgvector must be installed from the correct
PostgreSQL package repository or built from source.


8. KHAN ACADEMY SCRAPING
------------------------

Khan flow:

User submits Khan URL
-> app validates the URL
-> scrape job is created
-> worker fetches Khan course payload
-> worker maps course, units, lessons, videos
-> worker fetches each video content payload
-> worker tries Khan transcript/subtitles
-> if transcript exists, store transcript, timestamped segments, embeddings
-> if transcript does not exist, mark video no_transcript

Important fixes:
- Khan Academy timestamps were in milliseconds in some responses.
- The app originally treated them like seconds.
- This caused incorrect UI times.
- The fix converts millisecond values to seconds.

Example:
2000 ms becomes 2 seconds.
60000 ms becomes 60 seconds, displayed as 01:00.


9. YOUTUBE PLAYLIST SCRAPING
----------------------------

YouTube playlist support was added after the MVP.

Goal:
Allow URL like:
https://www.youtube.com/playlist?list=...

Flow:

User submits YouTube playlist URL
-> URL validator detects source_type youtube_playlist
-> worker uses yt-dlp to read playlist metadata
-> creates one course for the playlist
-> creates one unit/lesson to hold playlist videos
-> creates each video row
-> tries to fetch transcript once
-> stores transcript in database forever if successful
-> marks no_transcript if it fails

Important:
The app does not download video files.
It only attempts to fetch transcript/caption text.

Files:
- app/services/urls.py
- app/services/youtube_playlist_client.py
- app/services/youtube_client.py
- app/services/job_processor.py


10. YOUTUBE TRANSCRIPT HANDLING
-------------------------------

The user requested this strategy:

Fetch transcript once
-> store in database
-> serve from database forever

That is how the app is designed.

The app attempts:
- youtube-transcript-api first.
- Available transcript list fallback.
- yt-dlp/timed-text fallback.

Why some videos still show no transcript:
- YouTube may show "Show transcript" in a browser but block backend/server
  requests.
- Cloud IPs, including Azure VM IPs, can get HTTP 429 or blocked.
- Some videos do not expose captions through the backend-accessible endpoint.
- The UI transcript panel and backend transcript endpoint are not guaranteed
  to behave identically.

Safety:
- Fetching captions/transcripts does not harm the device.
- Public deployment risk is mostly reliability and terms/endpoint stability,
  not device damage.

faster-whisper fallback:
- Admin-triggered audio transcription was implemented for YouTube-backed videos
  that still have no transcript after the normal caption fetch attempts.
- The video page shows a Generate transcript form when a missing-transcript
  video has a YouTube ID.
- Admin enters ADMIN_KEY and queues a background job.
- The worker downloads audio with yt-dlp, extracts audio with ffmpeg, runs
  faster-whisper locally, stores transcript text and timestamped segments, and
  creates embeddings.
- The generated transcript source is stored as whisper_generated.
- All users then read/search the saved transcript from the database forever.
- This is intentionally queued in the worker instead of running inside the web
  request because transcription can be slow and CPU-heavy.

server {
    listen 80;

19. DEPLOYMENT ISSUES DEBUGGED
------------------------------

Docker/Podman issue:
- Docker command was being emulated by Podman.
- Port forwarding behaved incorrectly.
- Container internal health worked, but host port timed out.
- We decided to avoid Docker and use direct VM deployment.

Port confusion:
- Local dev UI was at:
  http://127.0.0.1:8000
- VM internal API was at:
  http://127.0.0.1:9000
- Public VM UI was:
  http://20.219.139.254

Nginx must match API service:
- If service uses 9000, Nginx proxy_pass must use 9000.
- If service uses 8000, Nginx proxy_pass must use 8000.

Missing dependency:
VM logs showed:
ModuleNotFoundError: No module named 'yt_dlp'

Fix:
cd ~/grammar
source .venv/bin/activate
pip install -r requirements.txt
sudo systemctl restart grammar-api grammar-worker

Admin key issue:
VM logs showed:
POST /api/categories 401 Unauthorized

Meaning:
UI admin key did not match ADMIN_KEY in /home/ubuntu/grammar/.env.

Fix:
- Open .env on VM.
- Copy only the value after ADMIN_KEY=.
- Paste that exact value into UI Admin key box.
- Do not paste ADMIN_KEY= itself.

pgvector missing:
VM logs showed:
extension "vector" is not available

Meaning:
- App still runs.
